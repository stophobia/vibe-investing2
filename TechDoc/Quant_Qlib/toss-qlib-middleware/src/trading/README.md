# 트레이딩(주문 실행) 모듈 — 미구현, 확장 지점

이 미들웨어는 **인증(OAuth2)과 시세/종목 데이터 조회까지만** 구현한다. 주문 생성·정정·취소, 잔고/체결 조회 등 실거래 기능은 의도적으로 포함하지 않았다. 이 디렉토리는 실제 주문 로직을 붙이고 싶은 사람이 확장하는 지점으로 비워둔 것이다.

확장 시 참고할 부분:

- 인증은 [`../auth/tokenService.ts`](../auth/tokenService.ts)의 `TossAuthService`를 그대로 재사용한다. 토큰 캐싱/재발급 로직을 다시 만들 필요 없다.
- HTTP 클라이언트는 [`../market/tossClient.ts`](../market/tossClient.ts)의 `TossApiClient.get` 패턴을 따라 `post`/`patch`/`delete` 메서드를 추가해 주문 엔드포인트를 감싸면 된다.
- 계좌·주문 관련 API는 공식 문서 기준 `Authorization` 헤더 외에 `X-Tossinvest-Account`(계좌 시퀀스번호) 헤더가 추가로 필요하다.
- 실주문 코드를 작성한다면 반드시 소액/모의 환경에서 먼저 검증하고, 주문 상태(대기/체결/부분체결/취소)를 추적하는 상태머신과 재시도 정책을 신중히 설계할 것. 네트워크 재시도 중 중복 주문이 나가지 않도록 idempotency key 처리가 필요하다.
- 이 디렉토리에는 의도적으로 코드가 없다.
