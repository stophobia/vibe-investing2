## 1. Resend

### 장점 (Pros)

- **뛰어난 개발자 경험**: 직관적인 API와 TypeScript SDK, React Email 라이브러리와 완벽히 통합되어 React 기반 환경에서 탁월한 생산성을 제공합니다.
- **Vercel과의 완벽한 궁합**: Vercel 출신 엔지니어들이 설계하여 Next.js 등 현대적인 JavaScript 스택에 최적화되어 있습니다.
- **실용적인 무료 플랜**: 월 3,000건 / 일 100건을 제공하는 현존하는 가장 실용적인 영구 무료 플랜입니다.
- **현대적인 기능**: 2025년에 수신 이메일 처리(inbound email) 기능이 추가되어 발신과 수신이 모두 가능합니다.
- **React Email 통합**: JSX로 이메일 템플릿을 작성하고 미리보기까지 제공하는 `react-email` 패키지와 공식 통합됩니다.

### 단점 (Cons)

- **SMTP 미지원**: 오직 HTTP API만 지원하여, 레거시 시스템 마이그레이션 시 어려움이 있습니다.
- **낮은 Rate Limit**: 기본적으로 초당 2건의 요청만 허용합니다. 여러 요청이 동시에 발생할 경우 제한에 쉽게 걸릴 수 있습니다.
- **마케팅 기능 부재**: 이메일 마케팅(뉴스레터)은 별도 상품으로 분리되어 있어 기본 API로는 대량 마케팅 메일 발송이 어렵습니다.
- **전담 IP 미제공**: 유료 플랜에서도 전용 IP 옵션이 없어 발신 IP가 동적으로 변경되어 평판 관리가 어렵습니다.
- **고급 마케팅 기능 부족**: A/B 테스팅, 복잡한 워크플로우 자동화 등이 부족합니다.
- **상대적으로 높은 유료 단가**: Pro 플랜은 월 $20부터 시작하며, 대량 발송 시 1,000건당 $0.90으로 경쟁 서비스 대비 가격 경쟁력이 떨어집니다.

### 빠른 시작 (Next.js 기준)

```bash
npm install resend
```

```typescript
// app/api/send/route.ts
import { Resend } from 'resend'

const resend = new Resend(process.env.RESEND_API_KEY)

export async function POST() {
  const { data, error } = await resend.emails.send({
    from: 'noreply@yourdomain.com',
    to: 'user@example.com',
    subject: '안녕하세요',
    html: '<p>Resend로 발송된 이메일입니다.</p>',
  })

  if (error) return Response.json({ error }, { status: 500 })
  return Response.json(data)
}
```

### 최종 평가

Resend는 Vercel과 Next.js로 구축된 최신 기술 스택의 트랜잭셔널 이메일에 최적화되어 있습니다. 개발자 경험을 최우선으로 하며, 무료 플랜의 실용성도 뛰어납니다. 하지만 마케팅 메일 발송, SMTP 필요성, 대규모 트래픽 처리가 중요하다면 다른 옵션을 고려해야 합니다.

## 2. Brevo (구 Sendinblue)

### 장점 (Pros)

- **가장 관대한 영구 무료 플랜**: 하루 300건 / 월 약 9,000건을 발송할 수 있는 현존하는 무료 플랜 중 가장 강력한 혜택을 제공합니다. 신용카드 등록도 필요 없습니다.
- **올인원(All-in-One) 플랫폼**: 트랜잭셔널 이메일과 마케팅 이메일을 하나의 플랫폼에서 통합 관리할 수 있습니다. SMS 기능까지 제공합니다.
- **경쟁력 있는 유료 가격**: 5,000명의 연락처를 보유한 경우 Mailchimp 대비 월 비용을 크게 절감할 수 있습니다.
- **다양한 기능**: 드래그 앤 드롭 이메일 빌더, CRM, A/B 테스트, 마케팅 자동화 등 강력한 부가 기능을 제공합니다.
- **AI 기능 내장**: 'Aura AI' 도우미를 통해 텍스트 및 이미지 생성을 지원합니다.
- **SMTP 지원**: API 외에도 SMTP를 지원하여 레거시 시스템 통합이 수월합니다.

### 단점 (Cons)

- **무료 플랜 브랜딩**: 무료 플랜을 사용할 경우, 발신되는 모든 이메일 하단에 Brevo 로고 및 광고 문구가 포함됩니다.
- **연락처 수 과금 변경**: '언리미티드' 정책 중단으로 인해 연락처 수에 따른 과금 체계로 변경되어 발송량이 많은 경우 비용이 증가할 수 있습니다.
- **성능 이슈**: 가끔 페이지 로딩이 느리거나 데이터 가져오기(import) 과정에서 문제를 겪을 수 있습니다.
- **통합의 한계**: 대규모 CRM과의 통합 측면에서 경쟁사 대비 다소 빈약할 수 있습니다.
- **개발자 경험**: Resend 대비 API가 다소 전통적이며, React와 같은 현대적인 프레임워크와의 통합이 깔끔하지 않습니다.

### 빠른 시작 (Node.js 기준)

```bash
npm install @getbrevo/brevo
```

```typescript
import * as SibApiV3Sdk from '@getbrevo/brevo'

const apiInstance = new SibApiV3Sdk.TransactionalEmailsApi()
apiInstance.setApiKey(
  SibApiV3Sdk.TransactionalEmailsApiApiKeys.apiKey,
  process.env.BREVO_API_KEY!
)

await apiInstance.sendTransacEmail({
  sender: { name: 'My App', email: 'noreply@yourdomain.com' },
  to: [{ email: 'user@example.com' }],
  subject: '안녕하세요',
  htmlContent: '<p>Brevo로 발송된 이메일입니다.</p>',
})
```

### 최종 평가

Brevo는 예산이 중요하고, 마케팅과 트랜잭셔널 이메일을 모두 처리해야 하는 중소기업 및 스타트업에 이상적입니다. 방대한 무료 플랜 덕분에 비용 부담 없이 시작할 수 있으며, 성장함에 따라 자연스럽게 유료 플랜으로 전환할 수 있습니다.

## 3. Mailgun

### 장점 (Pros)

- **안정적인 인프라**: 수십 년간 검증된 이메일 인프라로 대규모 트랜잭셔널 발송에 강점을 가집니다.
- **SMTP 및 API 모두 지원**: 레거시 시스템 통합과 현대적 API 방식을 모두 선택할 수 있습니다.
- **고급 라우팅 및 이메일 파싱**: 수신 이메일을 파싱하여 웹훅으로 전달하는 강력한 라우팅 기능을 제공합니다.
- **전용 IP 제공**: 유료 플랜에서 전용 IP를 사용할 수 있어 발신 평판 관리가 용이합니다.
- **이메일 유효성 검사 API**: 이메일 주소 형식 및 도메인 유효성 검사 API를 별도로 제공합니다.

### 단점 (Cons)

- **무료 플랜 제한적**: 일 100건의 영구 무료 발송은 타 서비스 대비 적습니다.
- **UI/UX 노후화**: 대시보드가 다소 구식이며 직관성이 떨어집니다.
- **마케팅 기능 부재**: 순수 트랜잭셔널 이메일 서비스로, 마케팅 캠페인 기능은 없습니다.
- **Flex 플랜 구조**: 사용량에 따라 비용이 변동되어 예산 예측이 어려울 수 있습니다.

### 최종 평가

기존에 SMTP 기반 시스템을 운영 중이거나, 이메일 파싱 및 고급 라우팅이 필요한 팀에 적합합니다. 대규모 트랜잭셔널 발송 인프라로는 여전히 업계 표준 수준입니다.

## 4. MailerSend

### 장점 (Pros)

- **직관적인 UI**: 드래그 앤 드롭 템플릿 빌더와 깔끔한 대시보드를 제공합니다.
- **영구 무료 플랜**: 월 500건을 영구적으로 무료 발송할 수 있습니다.
- **트랜잭셔널에 집중**: 발송 속도와 안정성에 최적화된 서비스입니다.
- **다양한 SDK**: JavaScript, PHP, Python, Ruby 등 주요 언어 SDK를 제공합니다.
- **이메일 미리보기**: 템플릿 편집 시 다크/라이트 모드 및 모바일 미리보기를 지원합니다.

### 단점 (Cons)

- **무료 플랜 발송량 적음**: 월 500건은 실 서비스 트래픽에 부족할 수 있습니다.
- **마케팅 기능 제한적**: Brevo 대비 올인원 마케팅 기능이 부족합니다.
- **레퍼런스 상대적으로 적음**: Resend나 SendGrid 대비 국내외 레퍼런스와 커뮤니티가 작습니다.

### 최종 평가

사용 편의성을 중시하는 팀, 특히 비개발자가 템플릿을 직접 편집해야 하는 환경에 적합합니다.

## 5. Amazon SES

### 장점 (Pros)

- **초저렴한 비용**: 1,000건당 $0.10의 비용으로 업계 최저 수준의 단가를 자랑합니다.
- **AWS 생태계 통합**: Lambda, SNS, S3 등 AWS 서비스와 긴밀하게 통합됩니다.
- **무제한 확장성**: AWS 인프라를 기반으로 수억 건의 발송도 안정적으로 처리합니다.
- **전용 IP 제공**: 추가 비용으로 전용 IP를 사용할 수 있습니다.

### 단점 (Cons)

- **높은 진입 장벽**: IAM 설정, 도메인 검증, 샌드박스 해제 등 초기 설정이 복잡합니다.
- **무료 플랜 기간 제한**: 12개월 무료 티어(월 3,000건) 이후 과금이 시작됩니다.
- **낮은 사용자 경험**: 대시보드가 기술적이며 이메일 마케팅 기능이 전혀 없습니다.
- **샌드박스 제한**: 기본적으로 샌드박스 모드로 시작하여, 프로덕션 전환을 위해 AWS 심사가 필요합니다.
- **낮은 Trustpilot 평점**: 지원 품질에 대한 불만이 많습니다.

### 최종 평가

AWS를 이미 사용 중이고 대규모 발송이 필요한 팀에 최적입니다. 초기 설정 복잡성을 감수할 수 있다면 장기적으로 가장 저렴한 선택입니다.

## 6. Mailtrap

### 장점 (Pros)

- **개발/테스트 환경 특화**: 실제 수신자에게 이메일을 보내지 않고 테스트할 수 있는 이메일 샌드박스를 제공합니다.
- **프로덕션 발송 통합**: 샌드박스와 프로덕션 발송을 하나의 플랫폼에서 관리할 수 있습니다.
- **팀 협업**: 여러 팀원이 동일한 이메일 테스트 환경을 공유할 수 있습니다.
- **영구 무료 플랜**: 일 150건을 영구적으로 발송 가능합니다.

### 단점 (Cons)

- **발송량 제한**: 무료 플랜의 일 150건은 실 서비스에 부족합니다.
- **주력이 테스팅**: 프로덕션 발송보다 개발 단계 테스팅에 더 특화되어 있습니다.
- **상대적으로 낮은 인지도**: 다른 서비스 대비 레퍼런스가 적습니다.

### 최종 평가

로컬 개발 및 QA 환경에서 이메일 테스트가 잦은 팀에 특히 유용합니다. 프로덕션 발송량이 적다면 단일 플랫폼으로도 활용 가능합니다.

## 7. SendGrid

### 장점 (Pros)

- **업계 최대 인프라**: Twilio가 인수한 이후 대규모 엔터프라이즈 고객을 다수 보유하고 있습니다.
- **풍부한 레퍼런스**: 가장 많은 서드파티 통합과 커뮤니티 자료를 가지고 있습니다.
- **마케팅 + 트랜잭셔널 통합**: 두 가지 기능을 모두 제공합니다.
- **고급 분석**: 발송 통계, 클릭률, 오픈율 등 상세한 분석 데이터를 제공합니다.

### 단점 (Cons)

- **영구 무료 플랜 없음**: 60일 체험판(일 100건) 이후 반드시 유료 전환이 필요합니다.
- **낮은 Trustpilot 평점(1.2/5)**: 고객 지원 품질과 계정 정지 관련 불만이 매우 많습니다.
- **계정 정지 이슈**: 명확한 사전 경고 없이 계정이 정지되는 사례가 보고됩니다.
- **복잡한 가격 체계**: 기능에 따라 가격 체계가 복잡합니다.

### 최종 평가

레퍼런스는 풍부하지만, 영구 무료 플랜이 없고 고객 지원 이슈가 많아 소규모 프로젝트나 스타트업에는 권장하지 않습니다. 이미 엔터프라이즈 계약을 체결한 경우에 적합합니다.

## 8. Postmark

### 장점 (Pros)

- **빠른 전송 속도**: 트랜잭셔널 이메일 발송 속도가 업계 최고 수준으로 알려져 있습니다.
- **높은 전달률**: 트랜잭셔널 이메일에 집중하여 스팸 처리 비율이 낮습니다.
- **명확한 로그**: 이메일 활동 로그가 상세하고 검색이 용이합니다.
- **영구 무료 플랜**: 월 100건을 영구적으로 무료 발송 가능합니다.

### 단점 (Cons)

- **무료 발송량 매우 적음**: 월 100건은 실질적으로 개인 프로젝트 수준에 그칩니다.
- **마케팅 기능 없음**: 순수 트랜잭셔널에만 집중하여 마케팅 캠페인 기능이 없습니다.
- **상대적으로 높은 유료 단가**: 소규모 플랜의 비용 효율이 낮습니다.

### 최종 평가

전송 속도와 전달률이 비즈니스 핵심 지표인 서비스(이커머스 주문 알림, 금융 거래 알림 등)에 적합합니다.

## 10. 선택 가이드 및 최종 결론

### 시나리오별 추천

**개인 프로젝트 / 사이드 프로젝트 / 개발자 경험 최우선**  
→ **Resend**: 초기 설정이 가장 쉽고 코드 작성이 편리합니다. 무료 플랜(월 3,000건)으로 대부분의 개인 프로젝트는 문제없이 감당 가능합니다.

**마케팅 메일과 트랜잭셔널 메일을 함께 처리해야 하는 스타트업**  
→ **Brevo**: 무료 플랜(일 300건)이 가장 넉넉하고 마케팅 자동화 등 부가 기능이 뛰어납니다. 향후 성장 시 유료 전환해도 비용 부담이 적습니다.

**기존 SMTP 기반 시스템 통합이 필요한 경우**  
→ **Mailgun**: SMTP를 지원하므로 기존 시스템과의 통합이 수월합니다. 안정성도 검증된 서비스입니다.

**AWS 생태계에 이미 투자했거나 극단적인 비용 효율이 필요한 경우**  
→ **Amazon SES**: 다른 서비스 대비 현저히 저렴한 비용이 장점입니다. 다만 12개월 이후 과금 시작과 초기 설정 복잡성은 감안해야 합니다.

**로컬 개발 및 QA 테스트 환경이 중요한 팀**  
→ **Mailtrap**: 샌드박스 환경에서 실제 발송 없이 이메일을 테스트할 수 있어 개발 생산성이 크게 향상됩니다.

**전송 속도와 전달률이 핵심 지표인 서비스 (이커머스, 금융 등)**  
→ **Postmark**: 트랜잭셔널에 특화된 빠른 전송 속도와 높은 전달률이 강점입니다.

### Vercel + Next.js 환경 종합 추천

Vercel 환경을 기준으로 전반적으로는 **Resend**가 가장 최적의 선택입니다. Vercel과의 궁합, React/Next.js와의 통합, 무료 플랜의 실용성 모두 우수하기 때문입니다.

다만, 아래 조건에 해당한다면 대안을 고려하세요.

- 마케팅 이메일이 사업 핵심이라면 → **Brevo**
- SMTP가 반드시 필요하다면 → **Mailgun** 또는 **Brevo**
- 비용을 극단적으로 줄여야 한다면 → **Amazon SES**
- 발송 전달률이 SLA에 영향을 준다면 → **Postmark**

### 도메인 인증 공통 주의사항

어떤 서비스를 선택하더라도 아래 DNS 레코드 설정은 필수입니다. 미설정 시 스팸 처리 비율이 급격히 높아집니다.

| 레코드 | 역할 | 설정 위치 |
|--------|------|----------|
| **SPF** | 발송 서버 인증 | DNS TXT 레코드 |
| **DKIM** | 이메일 서명 위변조 방지 | DNS TXT 레코드 |
| **DMARC** | SPF/DKIM 정책 선언 | DNS TXT 레코드 |
| **Custom Domain** | 발신자 주소 브랜딩 | 각 서비스 대시보드 |

> SPF, DKIM, DMARC 세 가지를 모두 설정해야 Gmail, Outlook 등 주요 이메일 클라이언트에서 스팸으로 분류되지 않습니다. 특히 2024년부터 Google과 Yahoo는 대량 발송자(일 5,000건 이상)에게 DMARC 설정을 의무화하고 있습니다.

## 부록. 한국 서비스 환경 고려사항

- **네이버 메일**: 외부 이메일 서비스 발송 시 스팸 처리 비율이 높습니다. 도메인 인증 3종(SPF/DKIM/DMARC) 완비가 필수입니다.
- **카카오 계정**: `@kakao.com` 수신자에 대한 대량 발송은 별도 화이트리스트 신청이 필요할 수 있습니다.
- **국내 ISMS/개인정보보호법**: 이메일 수신 동의 기록 보관 의무가 있습니다. 발송 서비스 선택 시 수신 동의 로그를 별도로 관리하는 체계를 함께 구축해야 합니다.
- **데이터 레지던시**: 개인정보가 포함된 이메일을 발송하는 경우, 각 서비스의 데이터 처리 서버 위치(EU, US 등)를 확인하고 개인정보처리방침에 명시해야 합니다.
