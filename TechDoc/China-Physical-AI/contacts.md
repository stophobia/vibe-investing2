## 활용 가이드

1. **학술/기술 협업 문의**: Amap CV Lab, LimX Dynamics처럼 논문·README에 명시된 이메일이 있는 경우 가장 응답률이 높다. 제목에 논문명 또는 프로젝트명을 명시할 것.
2. **파트너십/투자 문의**: 앤트그룹·알리바바·텐센트 계열은 개별 연구조직이 아닌 그룹 IR·BD 채널을 통하는 것이 정석이며, 개별 연구자 이메일로의 직접 접촉은 권장되지 않는다.
3. **오픈소스 기여/버그 리포트**: 모든 프로젝트에서 GitHub Issues가 가장 신속한 채널이다. 이메일보다 Issue 등록 시 유지보수팀의 응답 속도가 빠른 경향이 관찰된다(Dexbotic, FluxVLA README에서 공식 권장).
4. **미확인 항목 검증 방법**: 회사명 + "official website" + "contact"로 재검색하거나, 해당 기업의 최신 보도자료(BusinessWire, PR Newswire, 중국어 매체)에 첨부된 언론 담당자 연락처를 활용할 것을 권장한다.
