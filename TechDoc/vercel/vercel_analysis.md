# Vercel 플랫폼 분석: 장점, 단점 및 리스크

## 1. 개요

Vercel은 Next.js 개발사가 제공하는 프론트엔드 배포 및 호스팅 플랫폼으로, 뛰어난 개발자 경험(DX)과 프레임워크와의 긴밀한 통합을 특징으로 합니다. 그러나 예측 불가능한 비용 구조와 벤더 종속성이라는 중요한 리스크도 함께 존재합니다.

본 문서는 Vercel 도입을 고려하는 팀과 개발자에게 기술적 의사 결정을 위한 객관적인 정보를 제공합니다.

---

## 2. 장점

### 2.1 탁월한 개발자 경험 및 생산성

- **Git 기반 자동 배포**: GitHub, GitLab, Bitbucket과 연결 시 메인 브랜치 푸시만으로 자동 배포되며, 각 기능 브랜치별 실시간 프리뷰 URL이 생성됩니다.
- **프레임워크 최적화**: Next.js의 모든 기능(App Router, 서버 컴포넌트, ISR, 이미지 최적화 등)이 기본적으로 지원됩니다. 또한 Nuxt, SvelteKit, Astro, Remix, Vue 등 40개 이상의 현대 프레임워크를 동등하게 지원합니다.
- **프레임워크 기반 인프라(FDI)**: 개발자가 Vercel 전용 API를 배우지 않아도 프레임워크 코드만으로 인프라를 자동 프로비저닝합니다.

### 2.2 글로벌 성능 및 통합 기능

- **엣지 네트워크**: 글로벌 CDN 기본 탑재, 엣지 미들웨어/함수를 통한 위치 기반 라우팅 및 인증 처리 가능
- **통합 툴체인**: 이미지 최적화, 자동 HTTPS, DDoS 보호, 애널리틱스, AI 도구(v0) 등을 단일 플랫폼에서 제공

## 3. 단점

### 3.1 예측 불가능한 비용 구조

Pro 요금제(월 $20/좌석) 기준 기본 제공량 초과 시 사용량 기반 과금이 적용됩니다.

| 항목 | 기본 제공량 | 초과 요금 |
|------|-------------|------------|
| 빌드 시간 | 6,000분/월 | 분당 $0.01 |
| 서버리스 함수 호출 | 100만 회/월 | 회당 과금 |
| 대역폭 | 1TB/월 | GB당 $0.15 |
| 이미지 최적화 | 5,000개 소스 이미지 | 1,000개당 $0.60 |

> ⚠️ 트래픽 증가 시 급격한 비용 상승 가능. 빌드 루프나 오류로 인해 예상치 못한 추가 비용이 발생할 수 있습니다.

### 3.2 백엔드 및 장시간 작업 지원 부족

- **지원되지 않는 작업**: 백그라운드 잡(주기적 작업, 큐), WebSocket 등 장시간 연결, 장시간 실행 프로세스
- **함수 실행 시간 제한** 존재
- 사용자 정의 런타임이나 특정 Node.js 버전 사용 제한

### 3.3 제한적인 디버깅 및 제어권

- 인프라 내부 상태 모니터링 및 디버깅 수단 부족
- 커스텀 프록시(예: Cloudflare)를 앞단에 둘 경우, Vercel Firewall과 DDoS 방어 기능이 정상 작동하지 않음

### 3.4 보안 및 서비스 불안정성

- DDoS 공격 시 대역폭 비용 급증 위험
- 고객 지원 응답 지연 및 배포 실패에 대한 불만 사례 존재 (Trustpilot 평점 1.8/5)
- 무료 요금제(Hobby)는 상업적 사용이 제한적이며, 공정 사용 가이드라인 위반 시 계정 정지 가능

## 4. 리스크 (Risks)

### 4.1 높은 벤더 종속성 (Vendor Lock-In)

| 종속 영역 | 설명 | 마이그레이션 영향 |
|------------|------|--------------------|
| `next/image` | Vercel 최적화 파이프라인에 의존 | 다른 환경에서 성능 저하 또는 기능 불가 |
| `next/link` | Vercel 엣지 네트워크의 프리페칭 활용 | 기본 프리페칭 동작 재구현 필요 |
| 서버 액션 (Server Actions) | HTTP 계약 없이 FE/BE 로직 혼합 | 모든 액션을 일반 API 엔드포인트로 분리하는 대규모 리팩토링 필요 |
| 캐싱 시스템 | Next.js 13+ 캐싱이 Vercel 인프라와 결합 | 다른 환경에서 동일한 성능 보장 어려움 |

> 📌 핵심: Next.js 고유 기능을 많이 사용할수록 이탈 비용이 기하급수적으로 증가합니다.

### 4.2 재정적 리스크 (예산 폭탄)

- 트래픽 증가에 따른 비용 예측 불가능
- 스타트업 및 개인 개발자에게 치명적일 수 있음
- 최근 오픈소스 PaaS(Dokploy, Coolify 등)로의 이주 사례 증가

### 4.3 비즈니스 연속성 리스크

- 공정 사용 가이드라인(Fair Use Guidelines), AI 생성 콘텐츠 규정, 계정 사용 제한 등 정책이 예고 없이 변경될 수 있음
- 정책 변경으로 인한 워크플로우 강제 전환 사례 존재 (예: v0의 브랜치 기반 워크플로우 도입)

## 5. 종합 평가 및 권장사항

| 유형 | 적합한 경우 | 부적합한 경우 |
|------|--------------|----------------|
| **프로젝트 성격** | Next.js 기반 마케팅 사이트, 문서 사이트, 개인/팀 블로그, 빠른 프로토타이핑 | 복잡한 백엔드, 장시간 작업, 백그라운드 잡이 필요한 앱 |
| **비용 민감도** | 취미 프로젝트(무료 티어), 트래픽이 매우 적은 상업용 앱 | 트래픽 변동이 크거나 예측 불가능한 SaaS, 엄격한 예산 관리 필요 조직 |
| **장기 전략** | 단기간 빠른 출시가 중요한 팀, 벤더 종속성을 감수할 수 있는 기업 | 장기적 마이그레이션 자유도가 중요한 프로젝트 |

### 최종 결론

Vercel은 **생산성과 개발 경험 측면에서 업계 최고 수준**이지만, 그 편리함은 **높은 비용 불확실성**과 **장기적인 벤더 종속성**이라는 댓가를 동반합니다.

- **스타트업/개인 개발자**: 초기에는 무료/저렴하더라도 성장 후의 예상 비용을 반드시 사전 계산
- **장기적 자유도와 비용 안정성이 중요하다면**: 자체 호스팅 PaaS(Dokploy, Coolify 등) 또는 Cloudflare Pages를 대안으로 고려
