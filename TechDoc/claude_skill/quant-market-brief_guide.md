# quant-market-brief 설치 및 사용 가이드

> 오늘의 주식 시황을 퀀트 관점(팩터·변동성·수급·레짐)으로 요약하는 Claude Skill
> 개념·설계 배경은 [Claude_skill_guide.md](./Claude_skill_guide.md) 5장 참조

---

## 1. 이 스킬이 하는 일

"오늘 시황 요약해줘" 한 마디로 뉴스 나열이 아닌 **구조화된 레짐 브리핑**을 생성합니다.

- **트리거 문구**: "시황", "오늘 장", "마켓 브리핑", "장 요약", "오늘 코스피/나스닥 어때" 등
- **출력 구조** (고정):
  1. 한 줄 레짐 판정 — `레짐: Risk-On|Risk-Off|Rotation|Chop (확신도: 상|중|하) — 근거`
  2. 숫자 테이블 (지수/변동성/금리/환율/수급, 출처·기준시각 명기)
  3. 팩터 스코어카드 (모멘텀·성장/가치·대형/소형·퀄리티 4축)
  4. "어제와 달라진 것" — 델타만 서술
  5. 반증 조건 — 판정이 틀렸다면 내일 관측될 지표
- **설계 원칙**: 웹 검색 강제(기억으로 답하지 않음), 예측 금지(조건부 시나리오만), 매매지시 금지

레짐 판정 줄은 고정 형식이므로 `portfolio-daily-review` 등 다른 스킬이 파싱해 재사용할 수 있습니다.

## 2. 설치 방법

세 가지 방법 중 하나를 선택하세요.

### 방법 A: Claude.ai (웹/앱) — 권장

1. 이 저장소의 스킬 폴더 전체를 zip으로 압축하거나, 배포된 `.skill` 파일을 준비합니다.
   ```bash
   # 폴더에서 직접 만들 경우 (zip = .skill과 동일 포맷)
   zip -r quant-market-brief.skill quant-market-brief/
   ```
2. Claude.ai → **Settings → Capabilities → Skills** 에서 업로드합니다. (유료 플랜)
3. 업로드 후 새 대화에서 트리거 문구를 말하면 자동 발동됩니다.

### 방법 B: Claude Code

개인 스킬 디렉토리에 폴더째 복사합니다:

```bash
cp -r quant-market-brief/ ~/.claude/skills/quant-market-brief/
```

Claude Code 재시작 후 자연어로 언급만 하면 됩니다.

### 방법 C: Claude API

[Skills API Quickstart](https://docs.claude.com/en/api/skills-guide#creating-a-skill)를 참고해
`.skill` 패키지를 업로드하면 API 호출에서도 동일하게 작동합니다.

> **검증 팁**: 설치 후 "지금 사용 가능한 스킬 알려줘"라고 물으면 로드 여부를 확인할 수 있습니다.

```markdown
# Quant Market Brief

## 목적

개별 뉴스가 아닌 시장 구조(레짐)를 읽는다. "무슨 일이 있었나"가 아니라
"팩터·변동성·수급 관점에서 시장이 어느 레짐에 있고, 어제와 무엇이 바뀌었나"에
답한다.

원칙: **관측된 사실 → 팩터 해석 → 조건부 시나리오**. 예측하지 않는다.

## 워크플로우

### 1단계: 데이터 수집 (웹 검색 필수)

다음을 반드시 웹 검색으로 확인한다. 학습 데이터의 기억으로 답하지 않는다.
모든 수치에 출처와 기준 시각을 명기한다.

- **주요 지수**: KOSPI, KOSDAQ, S&P 500, NASDAQ, 필라델피아 반도체지수(SOX)
  — 종가/등락률
- **변동성**: VIX 레벨과 전일 대비 변화율, VKOSPI
- **금리/환율**: 미국 10년물 국채금리, 원/달러 환율, 달러인덱스(DXY)
- **수급**: KRX 외국인/기관 순매수, 주요 섹터 ETF 자금 흐름
- **크립토(선택)**: BTC 가격, 김치프리미엄 — 위험선호 프록시로만 사용하고
  방향 신호로 직접 쓰지 않는다

사용자가 특정 시장(한국만, 미국만)을 지정하면 그 시장에 집중하되
글로벌 리스크 지표(VIX, 금리, 달러)는 항상 포함한다.

### 2단계: 팩터 렌즈 적용

`references/factor-checklist.md` 를 읽고 다음 4개 축을 판정한다:

1. **모멘텀 vs 리버설**: 최근 주도주가 계속 가는가, 꺾이는가
2. **성장 vs 가치**: 금리 방향과 스타일 로테이션의 정합성
3. **대형 vs 소형**: 위험선호가 소형주까지 확산되는가
4. **퀄리티/로우볼**: 방어 팩터로의 자금 이동 여부

각 축은 강세/중립/약세 + 근거 1줄로 판정한다.

### 3단계: 레짐 판정

아래 넷 중 하나로 명시적으로 분류하고 근거를 한 줄로 단다:

- **Risk-On**: 위험자산 광범위 상승, 변동성 하락, 소형주/크립토 동반 강세
- **Risk-Off**: 방어자산 선호, 변동성 급등, 안전통화 강세
- **Rotation**: 지수는 정체하나 섹터/스타일 간 자금 이동 뚜렷
- **Chop**: 방향성 부재, 낮은 거래대금, 팩터 신호 엇갈림

판정 확신도를 상/중/하로 표기한다. 데이터가 엇갈리면 "엇갈림"이라고
그대로 쓴다. 억지로 하나의 내러티브를 만들지 않는다.

### 4단계: 출력

`references/output-template.md` 의 템플릿을 따른다. 필수 포함 요소:

1. **한 줄 레짐 판정** (맨 위, 확신도 포함)
2. **숫자 테이블** (지수/변동성/금리/환율/수급)
3. **팩터 스코어카드** (4개 축, 각각 판정 + 근거 1줄)
4. **"어제와 달라진 것"** — 델타에 집중. 어제와 같은 얘기는 반복하지 않는다
5. **반증 조건** — 오늘 판정이 틀렸다면 내일 무엇이 관측될 것인가

## 가이드라인

- 예측하지 않는다. 조건부 시나리오("X가 유지되면 Y 레짐 지속")로만 서술한다.
- 모든 수치에 출처와 기준 시각을 명기한다.
- 투자 권유 표현("사라/팔아라/비중을 늘려라")을 쓰지 않는다. 판단 재료만 제공한다.
- 뉴스 헤드라인은 팩터 해석에 필요한 것만 인용한다. 헤드라인 나열은 금지.
- 장 마감 전 요청이면 "장중 데이터, 마감 후 변동 가능"을 명시한다.
- 다른 스킬(예: portfolio-daily-review)이 이 브리핑의 레짐 판정을 재사용할 수
  있으므로, 레짐 판정 줄은 항상 동일한 형식을 유지한다:
  `레짐: {Risk-On|Risk-Off|Rotation|Chop} (확신도: {상|중|하}) — {근거 1줄}`
```

### 4.2 references/factor-checklist.md

```markdown
# 팩터 점검 체크리스트

각 팩터는 **강세 / 중립 / 약세** 로 판정하고 근거를 1줄로 남긴다.
신호가 엇갈리면 "엇갈림"으로 표기하고 어떤 지표끼리 충돌하는지 쓴다.

## 1. 모멘텀 vs 리버설

- **점검 지표**: 최근 1개월 상위 수익률 섹터의 당일 상대수익률, 주도주 거래대금
- **모멘텀 강세**: 주도 섹터 상대수익률 > 시장 && 거래대금 유지 이상
- **모멘텀 붕괴 신호**: 주도주 급락 + 거래량 급증 (분산 매도 패턴)
- **리버설 신호**: 최근 낙폭과대 종목군의 광범위 반등 + 공매도 잔고 감소

## 2. 성장 vs 가치

- **점검 지표**: 미국 10년물 금리 방향, 성장주 지수(NASDAQ) vs 가치주 상대성과
- **정합성 체크**: 금리 상승 + 성장주 강세 같은 '비정합' 조합이 나타나면
  반드시 별도 표기 — 비정합은 레짐 전환의 조기 신호일 수 있다
- **주의**: 하루 데이터로 스타일 로테이션을 단정하지 않는다. 3~5거래일
  누적 방향을 함께 언급한다

## 3. 대형 vs 소형

- **점검 지표**: KOSPI vs KOSDAQ 상대성과, Russell 2000 vs S&P 500
- **위험선호 확산**: 소형주 지수가 대형주를 아웃퍼폼 + 거래대금 증가
- **위험선호 위축**: 대형주 쏠림 심화 (지수 상승이 소수 종목에 집중)
  — 상승장이어도 breadth 악화는 약세 신호로 별도 표기

## 4. 퀄리티 / 로우볼 (방어 팩터)

- **점검 지표**: 필수소비재/유틸리티/헬스케어 상대성과, 배당주 ETF 자금 유입
- **방어 이동 신호**: 지수 보합~상승인데 방어 섹터가 아웃퍼폼하면
  스마트머니의 선제 방어 가능성 표기

## 변동성 레짐 (모든 팩터 판정의 배경)

| VIX 레벨 | 레짐 | 함의 |
|---|---|---|
| < 15 | 저변동성 | 캐리/모멘텀 전략 유리, 단 급반전 취약 |
| 15–25 | 중립 | 팩터 신호 그대로 해석 |
| > 25 | 고변동성 | 로우볼/퀄리티 우위, 포지션 축소 검토 구간 |

**주의**: VIX '레벨'보다 '변화율'이 단기 신호로 유효하다.
전일 대비 +20% 이상 점프는 레벨과 무관하게 별도 경고로 표기한다.

## 수급 해석 원칙

- 외국인 순매수는 원/달러 방향과 함께 읽는다 (환헤지 플로우 구분)
- 기관 순매수는 프로그램/비프로그램을 가능하면 구분한다
- 하루 수급으로 추세를 말하지 않는다. 5거래일 누적과 함께 표기한다
```
