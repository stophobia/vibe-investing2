# AI Super Cycle — Weekly Dynamic Rebalancing Prompt (한국어판)

> **AI 슈퍼사이클 주간 동적 리밸런싱 전략** — `AI Super Cycle Prompt` 의 자매(sister) 전략.
> 본 프롬프트는 *종목 발굴*이 아니라, 이미 선별된 AI Value Chain 4-Layer 포트폴리오를
> **매주 위상(Phase)으로 라벨링**하고, **NVDA 사이클 + HBM 사이클로 동적 비중을 조절**하며,
> **MSFT·GOOGL(L2) + 현금성 자산(Cash)** 을 사이클에 맞춰 편입/축소하는 *실행 룰북*입니다.

경고: 이 프롬프트는 매우 고위험이며 매우 매우 리스크합니다.

---

## 사용 방법

1. 매주 같은 요일·시간에 아래 프롬프트 블록을 LLM(Claude / GPT / Gemini)에 붙여넣습니다.
2. `[이번 주 입력]` 섹션의 관측치를 최신 데이터로 갱신합니다.
3. LLM 이 위상 라벨 + 목표 비중표 + 이번 주 권장 액션(매수/매도/유지)을 출력합니다.
4. (선택) `ai_supercycle_rebalancer.py` 로 동일 로직을 코드로 재현·교차검증합니다.

> **자매 관계**: `AI Super Cycle Prompt` = 무엇을 담을지(분류·점수화) / 본 프롬프트 = 언제 얼마나 사고팔지(타이밍·비중).

## 핵심 설계

### 4위상 라벨링 (Phase Labeling)

매주 세 묶음의 신호를 종합해 **ACCEL / NEUTRAL / PEAK / DOWN** 중 하나로 라벨링합니다.

| 신호 묶음 | 구성 지표 | 가중 |
| --- | --- | --- |
| **Capex 2차미분** | 빅4(MSFT·GOOGL·AMZN·META) 합산 capex YoY 성장률의 *가속/감속* | 최대 |
| **HBM 사이클** | HBM 리드타임 추세 + 현물가 MoM | 중 |
| **NVDA 자체** | 데이터센터 매출 QoQ 추세 + 재고 MoM + GPU ASP QoQ | 중 |

> Capex 2차미분이 가장 신뢰도 높은 *선행지표*입니다. 절대 금액이 아니라 "증가율이 가속하는가, 감속하는가"를 봅니다.

### NVDA 동적 목표 비중 (밴드 + HBM 보정)

```
저점/가속(ACCEL)  →  목표 40%
중립(NEUTRAL)     →  목표 32.5%
정점(PEAK)        →  목표 28%
하강(DOWN)        →  목표 25%

[HBM 보정]  리드타임 단축 시 -2%p, 현물가 하락 시 -2%p
[ASP 보정]  GPU ASP 분기 -5%↓ 시 -3%p
[STRESS 캡] 시장 악화 또는 연준 인상 → NVDA 35% 강제 상한
```

### 매매 규칙 (A/B/C)

- **규칙 A (밴드 리밸런싱, 메인 엔진)**: NVDA 비중이 목표 밴드를 벗어나면, *이탈분의 1/3만* 이번 주 조정. 전량 금지 → 평균단가 분산.
- **규칙 B (저점 단계 매수)**: 하강·바닥 국면에서 NVDA 고점대비 **-30% → 1차**, **-45% → 2차** 단계 매수. 단, capex 명백한 감소 초입에는 **보류**(떨어지는 칼날 방지).
- **규칙 C (축소분 분산)**: NVDA에서 덜어낸 자금을 **L2(MSFT·GOOGL) 우선 → 충족 후 L3(전력·냉각·네트워크)** 로. 정점·하강일수록 분산 강화.

### 현금성 자산(Cash) 룰

```
평상시(ACCEL/NEUTRAL)  →  5%
정점(PEAK)              →  ~8.5%
하강(DOWN)              →  12%
시장 악화(STRESS)       →  20%
```

## 프롬프트 (한국어판)

```
[역할]
당신은 AI 인프라 사이클을 12년간 추적해온 퀀트 포트폴리오 매니저입니다.
당신의 임무는 종목 발굴이 아니라, 이미 선별된 AI Value Chain 4-Layer 포트폴리오를
매주 '위상(Phase)'으로 라벨링하고, NVDA 사이클 + HBM 사이클에 따라 동적으로
비중을 조절하는 것입니다. 절대 한 번에 전량 매매하지 않으며, 이탈분의 1/3씩
분할 조정하는 규율을 철저히 지킵니다.

현재 날짜: {YYYY-MM-DD}

[자산 유니버스 & Layer 매핑]
- L1 Foundation (선행·정점, 변동성 최상): NVDA(GPU), SNDK(스토리지), INTC
- L2 Infrastructure (중간·구조적): MSFT(Azure AI 현금흐름), GOOGL(GCP+TPU, 점유율 헤지)
- L3 Enablers (구조적 우상향, 평탄화): VRT/ANET/ETN 등 전력·냉각·네트워크
- L4 Application (후행): QQQ(나스닥100 간접)
- CASH: MMF / 단기국채 / SGOV류 현금성 자산

[이번 주 입력]   ← 매주 이 값만 갱신
- 빅4 합산 capex YoY 성장률: {예: 67}% (직전 관측: {62}%)
- HBM 리드타임: {20}주 (직전: {18}주), HBM 현물가 MoM: {+4}%
- NVDA 데이터센터 매출 QoQ: {22}% (직전 분기: {22}%)
- NVDA 재고 MoM: {+2}%, GPU ASP QoQ: {+1}%
- 시장 악화 여부: {false}, 연준 인상 전환 여부: {false}
- NVDA 고점대비 낙폭: {-8}%, 메모리 대표주 낙폭: {-10}%
- 현재 보유 비중(%): NVDA {52}, SNDK+INTC {16}, MSFT {0}, GOOGL {0}, QQQ {12}, CASH {0}

[수행 절차]
1. 위상 라벨링: 위 신호를 종합해 ACCEL / NEUTRAL / PEAK / DOWN 중 하나로 판정하고 근거 제시.
2. NVDA 동적 목표 비중 산출: 위상 앵커(40/32.5/28/25%) + HBM 보정 + ASP 보정 + STRESS 캡(35%).
3. 전체 목표 비중표 작성: NVDA / L1기타 / MSFT / GOOGL / L3 / QQQ / CASH, 합계 100%.
4. 이번 주 액션: 규칙 A(이탈분 1/3) + 규칙 C(축소분을 L2→L3 분산) + 규칙 B(급락 단계매수, 칼날 보류 점검).
5. 현금성 자산 목표 안내.

[출력 형식]
파트 1: 위상 라벨 + 신호별 점수(capex 2차미분 / HBM / NVDA QoQ / 정점경고) + 종합점수
파트 2: 목표 비중표 (현재 vs 목표 vs 이번주 조정 %p)
파트 3: 이번 주 실행 액션 리스트 (BUY/SELL/HOLD, 자산, %p, 사유)
파트 4: 모니터링 경고 (ASP -5%↓ / capex YoY +10%↓ / DC 공실률 5%↑ / 채권조달 부담)
파트 5: 다음 주 점검 포인트

[규율]
- 절대 한 번에 목표까지 매매하지 말 것. 이탈분의 1/3만.
- 시장 악화·연준 인상 시 NVDA 35% 캡 + 현금 상향을 최우선.
- capex 명백한 감소 초입에는 급락 매수(규칙B) 보류.
- 답변은 한국어로 작성.

[경고 문구]
본 분석은 교육·정보 제공 목적의 시뮬레이션이며 투자 자문이 아닙니다.
AI 슈퍼사이클은 닷컴버블(2000) 대비 펀더멘털이 강하나 Capex 정점 후 큰 폭의
드로다운이 가능합니다. 실제 매매 결정과 책임은 본인에게 있습니다.
```

## 한국 거주자 추가 고려 사항

- **양도소득세 22%** (250만원 공제 후): 규칙 A의 분할 매도는 *연 250만원 공제를 분산 활용*하기에 유리합니다.
- **현금성 자산**: 국내에서는 *KOFR 금리 추종 ETF(예: TIGER CD금리투자KIS)* 또는 *미국 SGOV* 로 현금 비중을 대체 가능. 환노출 여부는 원/달러 1,400원대 진입 시 환헤지 검토.
- **L2 편입**: MSFT·GOOGL 직접 보유 대신 *ISA 계좌 내 KODEX 미국나스닥100* 으로 간접 노출 시 절세 가능(단, 개별 비중 조절은 제약).

## 위험 고지

- 본 프롬프트의 결과는 *연구·교육용 시뮬레이션*이며 실제 투자 권유가 아닙니다.
- 위상 라벨링은 *입력 데이터의 정확도에 전적으로 의존*합니다 — capex/HBM/실적 수치를 신뢰 가능한 출처로 매주 갱신하십시오.
- 단일 종목(NVDA) 고집중은 사이클 정점에서 가장 위험합니다. 밴드 캡은 *신념과 리스크 관리의 절충*입니다.
- 능동적 매매 규칙의 최대 적은 "이번엔 다르다"며 규칙을 어기는 것입니다. 밴드·트리거를 *숫자로 미리 박아두는 것*이 핵심입니다.
- 실거래 전 반드시 독립 데이터 소스로 교차 검증하십시오.
