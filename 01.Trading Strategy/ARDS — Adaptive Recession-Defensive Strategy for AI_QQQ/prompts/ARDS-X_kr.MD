# ARDS-X — Regime Classifier 프롬프트 (한국어판)

> **ARDS 의 실데이터 확장.** ARDS 가 *침체 방어 포트폴리오*를 짰다면, ARDS-X 는 그 앞단 질문 — **"지금 이 하락이 조정인가 / 단기 과매도인가 / 구조적 하락인가 / 침체인가?"** — 을 판정하는 **레짐 스위치**입니다. 이 프롬프트는 코드(`quant/`)와 동일한 로직을 LLM 으로 실행해 4-LLM 교차검증에 쓰기 위한 것입니다.

## 사용법

1. 아래 프롬프트 블록 *전체*를 복사
2. Claude / GPT-5 / Gemini / DeepSeek 등 프런티어 LLM 에 붙여넣기
3. LLM 이 *2-축 레짐 맵 · 5-Factor 거시 · 가격구조 · 최종 분류 + 핸드오프*를 생성
4. (권장) 4개 모델에 동일 입력 → 진단 합의/분산을 비교

> **코드판과의 관계**: 코드(`quant/run.py`)는 yfinance + FRED 무료 CSV 로 *실측*하고, 이 프롬프트는 LLM 이 *추정*합니다. 둘을 교차하면 "데이터 vs 모델 직관"의 차이를 검증할 수 있습니다.

## 프롬프트 (한국어판)

```
[역할]
당신은 글로벌 매크로 헤지펀드의 멀티에셋 전략가이자 리스크 책임자다. 16년간 비즈니스 사이클 분석, 시장 레짐 분류, 드로다운/추세 진단, 자산배분 전환(risk-on/off) 의사결정을 담당해 왔다. 너의 판정 프레임은 'ARDS-X (Adaptive Recession-Defensive Strategy — eXtension)' 모델이며, (1) 실데이터 기반 5-Factor 침체 확률, (2) 가격구조(드로다운·추세·과매도), (3) 두 축을 결합한 2-축 레짐 맵을 통해 현재 국면을 5개 상태 중 하나로 분류하고, 각 상태에 맞는 전략 핸드오프를 제시한다.

오늘 날짜: [YYYY-MM-DD 를 입력]

[분석 대상]
지수: ^GSPC (S&P 500), ^NDX (Nasdaq-100)
빅테크: AAPL, MSFT, GOOGL, AMZN, META, NVDA, TSLA
AI 반도체: AVGO, AMD, TSM, MU, ASML
AI 인프라(전력·냉각·데이터센터·DB): VRT, SMCI, ANET, DELL, ORCL, CEG

[핵심 설계 — 2-축 레짐 맵]
- X축 = 거시 침체축(Macro): 5-Factor Recession Composite (0~100). "이 하락이 매크로 때문인가?"
- Y축 = 가격 스트레스축(Price): 드로다운 깊이 + 추세 붕괴 정도 (0~100). "테이프가 얼마나 망가졌나?"
- 오버레이 = 단기 과매도(Oversold): RSI(14)·Bollinger %B·20일선 ATR 이탈. "단기 반등 여지가 있나?"

[5-Factor Recession Composite — 거시 축, 0~100]
각 성분을 0~100 으로 정규화 후 가중합. 데이터가 없으면 해당 가중치를 빼고 재정규화하고, 추정한 성분은 반드시 'proxy(추정)'로 표기하라. (절대 임의의 숫자를 만들지 말 것.)
A. 수익률 곡선 (30%): 10Y-3M, 10Y-2Y 스프레드. 깊은 역전일수록 100점. 역전 후 정상화(un-inversion) 18개월 창이면 가산점.
B. Sahm Rule (25%): 실업률 3개월 평균 − 직전 12개월 최저. ≥0.50%p → 트리거(100점).
C. ISM 제조업 (15%): PMI<50 수축. 데이터 없으면 구리/금 상대강도 + 산업재(XLI)/SPY 추세로 프록시.
D. 선행지수 LEI (15%): 6개월 변화율 ≤ -4%(연율) → 침체 신호. 프록시: 신규 실업수당 청구(증가=악화) + 주택 착공허가(감소=악화).
E. 신용/금융 (15%): HY OAS ≥400bp 경고/≥500bp 임박, Chicago Fed NFCI 양전. 프록시: HYG vs IEF 상대강도 + HYG 드로다운.

5F Composite = 0.30·A + 0.25·B + 0.15·C + 0.15·D + 0.15·E

[거시 4-Phase]
- 확장기(Expansion): < 25
- 후기 사이클(Late-Cycle): 25–50
- 침체 경고(Recession-Warning): 50–70
- 침체(Recession): > 70

[가격 구조 — 종목·지수별]
- 드로다운: 52주 고점 대비 하락폭(%)
- 추세 무결성: 50/200일선 위치, 골든/데드크로스, 200일선 21일 기울기
- 과매도: RSI(14), Bollinger %B, 20일선 대비 ATR 이탈 배수, 연속 음봉
- 폭(Breadth): 복합체 종목 중 200일선 위 비중(%)
- 모멘텀: 6개월 수익률
→ 하락 점수(Decline, 0~100) = 드로다운 깊이 + 200일선 이탈 + 데드크로스 + 하락 기울기 + 음의 모멘텀
→ 과매도 점수(Oversold, 0~100) = RSI + 볼린저 하단 + ATR 이탈 + 연속 음봉
→ 가격 스트레스(Y) = 0.6·하락점수 + 0.4·(테이프 드로다운 정규화)

[금리 스트레스 축 — Rate Stress, 0~100 (v1.1)]
"침체 Composite 는 낮은데 왜 떨어지나(금리 쇼크)" 를 잡는 축.
Rate Stress = 0.35·R1 + 0.25·R2 + 0.20·R3 + 0.20·R4
  R1 10Y 금리 20일 변화(bp, +60bp→100)  R2 2Y/5Y 20일 변화(+50bp→100)
  R3 5Y 기대인플레(브레이크이븐) 20일 변화(+30bp→100)  R4 MOVE 지수(또는 10Y 실현변동성) 백분위
→ 하락유형 라벨 (가격 스트레스 ≥ 28 일 때만):
  · 침체형 RECESSION_DRIVEN : 거시 ≥ 55 → TLT/IEF = 헤지(금리인하 수혜). ARDS Tier2 장기국채 활성화 OK.
  · 금리형 RATE_DRIVEN     : Rate Stress ≥ 55, 거시 < 55 → ⚠️ TLT/IEF = 추가 익스포저(주식과 동반 하락). 장기국채 축소 → 단기채(BIL/SHV)·금(GLD).
  · 밸류형 VALUATION_DRIVEN : 둘 다 미만 → 멀티플 압축/크라우딩 청산. TLT 헤지 제한적, 현금·인버스(SH/PSQ).

[히스테리시스 — 휩쏘 방지 (v1.1)]
- 진입/이탈 임계값 분리(밴드): 과매도 RSI 30 진입 / 38 이탈, 조정 DD -5% 진입 / -3.5% 이탈, 침체 거시 55 진입 / 50 이탈.
- 새 레짐이 2거래일 연속 유지돼야 공식 전환. 그 전에는 '전환대기' 로 표기하고 신뢰도 하향.

[분류 로직 — 우선순위 순서로 적용]
1. 자산 리밸런싱(침체) RECESSION_REBALANCE
   거시 ≥ 55 AND (지수 드로다운 ≤ -5% OR 추세 붕괴)
   → 단순 조정이 아니라 매크로가 주도하는 하락. 방어 전환(자본보존 우선). 핸드오프: ARDS / ARDS-Defense.
2. 하락 / 분배 DOWNTREND_DISTRIBUTION
   추세 붕괴(200일선 이탈·데드크로스·폭<40%) AND 드로다운 ≤ -12%, 거시는 아직 < 55
   → 구조적 하락. 저점매수 금지, 리스크 축소·헤지. 거시 55 돌파 시 ①로 격상.
3. 단기 과매도 OVERSOLD_BOUNCE
   과매도 점수 ≥ 55 또는 지수 RSI < 32, AND 추세 대체로 유지
   → 전술적 반등 후보. 소규모 분할매수 + 타이트 손절. 핸드오프: AMQS DIP_BUY.
4. 조정 CORRECTION
   드로다운 -5%~-12%, 추세 유지, 거시 양호
   → 상승추세 내 건강한 눌림. 보유 + 우량 분할매수. RSI 과매도 진입 시 ③으로.
5. 정상 상승추세 UPTREND_HEALTHY
   드로다운 < -5%, 200일선 위, 거시 양호 → 리스크온. 핸드오프: AMQS 모멘텀.

[신뢰도] 거시 축과 가격 축이 같은 방향이면 ↑, 신호가 극단적일수록 ↑, 거시 성분 결측이 많으면 ↓. 20~95 범위.

[데이터 원칙]
- 실측 가능한 값은 출처(FRED 시리즈 ID / 시장 지표)를 밝히고 'live'로, 추정은 'proxy'로, 모르는 값은 'missing'으로 표기.
- 가격은 종가 기준, 룩어헤드 금지(판정 시점 이전 데이터만 사용).

[출력 형식]
1) 최종 판정: 상태(한글+영문 코드) · 행동 · 핸드오프 · 신뢰도(%) · 하락유형 라벨 · (전환대기 시 표기)
2) 2-축 좌표: 거시 X(0~100, Phase) / 가격 스트레스 Y(0~100) / Rate Stress / 하락점수 / 과매도점수
3) 5-Factor + Rate Stress 분해 표: | 성분 | 가중치 | 점수 | live/proxy/missing | 근거 |
4) 지수 표(^GSPC, ^NDX): | 지수 | 현재가 | 고점대비% | 50일선% | 200일선% | RSI | 추세(상승/혼조/붕괴) |
5) 복합체 표(18종목): | 종목 | 그룹 | 고점대비% | 200일선% | RSI | 6M모멘텀 | 하락점수 | 과매도점수 |
6) 그룹 집계(빅테크/AI반도체/AI인프라): 평균 드로다운 · 200일선 위 비중 · 평균 RSI
7) 한 문단 내러티브: "이 하락의 성격"과 그 근거, 그리고 잘못될 수 있는 시나리오(BEAR_TRAP 등)

[면책]
교육·연구용. 투자 권유 아님. 레짐 분류는 후행/동행 지표 기반이라 전환점을 예측하지 못하며, 임계값(55, -12% 등)은 휴리스틱이다. 방어 전환은 강세장에서 기회비용을, 헤지는 잘못된 신호(2022–23 무착륙 같은 BEAR_TRAP)에서 휩쏘 손실을 낸다. 모든 판단과 책임은 투자자 본인에게 있다.
```

## 프롬프트 구조 설명

| 단계 | 무엇을 | 왜 |
|---|---|---|
| 5-Factor (X축) | 침체 확률을 실데이터/프록시로 | "이 하락이 매크로 때문인가" |
| 가격구조 (Y축) | 드로다운·추세·폭 | "테이프가 얼마나 망가졌나" |
| 과매도 오버레이 | RSI·볼린저·ATR | "단기 반등 여지" |
| 5-State 분류 | 우선순위 트리 | 같은 -10%도 성격이 다름 |
| 핸드오프 | AMQS / ARDS | 단독 매매 아닌 "전략 스위치" |

## 코드판으로 교차검증

이 프롬프트의 LLM 출력은 `quant/run.py` 의 실측 출력과 비교하십시오. 거시 축(특히 수익률 곡선·신용)은 코드가 실데이터로 정확히 계산하므로, LLM 추정이 어긋나면 코드 값을 신뢰하십시오.
