## 0. TL;DR

포인트가 부족한 사용자가 **USDT 도네이션**으로 즉시 포인트를 충전할 수 있는 채널을 추가합니다. 스테이킹(환불 가능)과 달리 **환불 불가능한 일방적 후원**이며, 그 대가로 스테이킹 보너스의 **3~15배**에 해당하는 포인트를 즉시 지급합니다. **$100부터 배수가 급격히 증가**하여 고래 후원자에게 큰 가치를 제공합니다.

```
도네이션 = 환불 없음 → 큰 보상 (스테이킹의 3~15배)
스테이킹 = 환불 있음 → 작은 보상 (1,000 P/$100 + 일일 30~50 P/$100)
$1 체험   = 환불 없음 → 1일 프리미엄만
```

## 1. 도네이션 vs 스테이킹 vs 구독 비교

| 차원 | 도네이션 (신규) | 스테이킹 | $1 체험 | 정기 구독 |
|---|---|---|---|---|
| **환불성** | ❌ 환불 없음 | ✅ 100% 원금 환불 (7~35일) | ❌ 환불 없음 | 부분 환불 (정책에 따라) |
| **즉시 포인트** | ✅ 즉시 지급 | ✅ 1,000 P/$100 + 일일 적립 | ❌ | ❌ |
| **이자** | ❌ | ✅ 30~50 P/$100/일 | ❌ | ❌ |
| **프리미엄 활성화** | ✅ Point로 교환 가능 | ✅ 활성 동안 자동 | ✅ 1일 | ✅ 30일 |
| **최소 금액** | $1 | $100 | $1 | $10 |
| **목적** | 후원 + 포인트 충전 | 자산 보존 + 멤버십 | 시험 사용 | 정기 이용 |
| **지급 보너스 멀티플라이어** | **3~15× 스테이킹 환영 보너스** | 1× | — | — |

### 1.1 도네이션을 별도로 두는 이유

- **포인트 부족 사용자의 우회로**: 시간(미션·초대)을 들이지 못하는 사용자가 즉시 포인트 확보
- **환불 없는 후원이므로 운영자 수익 100%**: 스테이킹은 이자만 마진, 도네이션은 전액 매출
- **고래 후원자 친화 설계**: $5,000 도네이션은 $5,000 스테이킹보다 **15배 가치 있는 즉시 보상** — 큰 후원을 유인
- **법적 분류 명확화**: "후원·기부" 분류로 가상자산 사업자(VASP) 신고 부담 회피 (§9 참조)

## 2. 도네이션 티어 표

### 2.1 티어별 즉시 지급 포인트

스테이킹 환영 보너스 기준선 = **10 P/$1** (= 1,000 P/$100). 도네이션은 이 기준선의 3~15배.

| # | USDT | 멀티플라이어 | 즉시 지급 포인트 | 프리미엄 가치 환산¹ | 권장 사용 사례 |
|---|---|---|---|---|---|
| 1 | **$1** | 3× | **30 P** | 1일 미만 | 호기심 후원 |
| 2 | **$10** | 3× | **300 P** | 1일 미만 | 가벼운 응원 |
| 3 | **$50** | 4× | **2,000 P** | 4일 프리미엄 | 일반 후원 |
| 4 | **$100** | 7× ⚡ | **7,000 P** | 14일 프리미엄 | **급격한 도약점** |
| 5 | **$500** | 10× | **50,000 P** | 100일 프리미엄 | 골드 후원자 |
| 6 | **$1,000** | 12× | **120,000 P** | 240일 프리미엄 | 플래티넘 후원자 |
| 7 | **$3,000** | 14× | **420,000 P** | 800일 프리미엄 | 다이아몬드 후원자 |
| 8 | **$5,000** | 15× | **750,000 P** | 1,500일 (≈4년) 프리미엄 | 챔피언 후원자 |

¹ 프리미엄 30일권 = 15,000 P 기준 (§3.2 of v2.0 spec)

### 2.2 멀티플라이어 곡선 — $100 급격한 도약 설계

```
멀티플라이어 (×)
                                                  ●15× ($5K)
   15 ┤                                       ●14× ($3K)
   13 ┤                                  ●12× ($1K)
   11 ┤                             ●10× ($500)
    9 ┤
    7 ┤                    ●7× ($100) ◀── 급격 도약점
    5 ┤
    3 ┤  ●3× ●3× ●4×
    1 ┤  $1  $10 $50 ↑
       └─────────────┴──────────────────────────────→ USDT
                   $100부터 멀티플라이어가 4×→7× (1.75배 증가)
```

### 2.3 티어 선택 근거

- **$1~$50 (3~4×)**: 기본 후원층. 스테이킹 대비 약간 유리하지만 큰 차이 없음. 신뢰 구축 단계.
- **$100 (7×)**: **급격한 도약**. $100부터 시간 후원자는 "차라리 도네이션이 더 이득"이라고 느낌. 의도된 행동 유도점.
- **$500~$1,000 (10~12×)**: 골드/플래티넘 등급. 한국 retail 평균 자산층의 합리적 후원 상한선.
- **$3,000~$5,000 (14~15×)**: 챔피언 티어. 명예의 전당(§17.4 main spec) 등재 + 영구 다이아몬드.

### 2.4 도네이션 누적에 따른 명예 등급

도네이션 누적 합계로 별도 명예 라벨을 부여 (§4 티어와 별개):

| 누적 도네이션 | 명예 라벨 | 특전 |
|---|---|---|
| $100+ | 💝 Supporter | 프로필 핀 + 미니앱 후원자 갤러리 등재 |
| $500+ | 🥈 Patron | + 영구 골드 티어 보장 |
| $1,000+ | 🥇 Benefactor | + 영구 플래티넘 + 1:1 운영자 미팅 1회 |
| $5,000+ | 💎 Champion | + 영구 다이아몬드 + 명예의 전당 + 전용 페르소나 추천 1회 |

### 3.2 Multi-sig 보관 정책

스테이킹 핫월렛(§7.3 of v2.0 spec)과 **분리된 별도 multi-sig 지갑**을 사용합니다. 이유:

- 스테이킹 자금 = 사용자 환불 의무 있는 **타인 자산** → 100% custodial 보호
- 도네이션 자금 = 환불 의무 없는 **운영자 자산** → 운영비·마케팅 등에 사용 가능

| 지갑 종류 | 주소 정책 | Multi-sig | 용도 |
|---|---|---|---|
| 스테이킹 핫월렛 | §7.3 of v2.0 spec | 2/3 | 사용자 USDT 출금 대비 |
| **도네이션 수금 지갑** | **위 명시 주소** | 2/3 (별도 키) | 운영비 + 인프라 비용 |
| 도네이션 콜드월렛 | (별도 주소, 비공개) | 2/3 | 90% 보관 |

## 4. 입금 검증 흐름 — TON / TRC20 동시 처리

§7.4~§7.5 of v2.0 spec의 검증 인프라를 재사용합니다. 차이점만 기술:

### 4.1 TON 도네이션 검증

```python
# function_app.py에 추가
@app.schedule(schedule="0 */1 * * * *", arg_name="t")  # 매 1분 (스테이킹 2분보다 더 빠르게)
async def detect_ton_donations(t: func.TimerRequest):
    """운영자 도네이션 TON 지갑에 신규 USDT 입금 감지"""
    DONATION_TON_ADDR = "UQBt20bDOUdAyTYPi-L3rsEj7wBf95yBq0Bo5MjRGE-3ScqV"
    last_lt = await _read_last_processed_lt("ton_donation")

    txs = await _ton_api.get_jetton_transfers(
        address=DONATION_TON_ADDR,
        jetton_master=TON_USDT_JETTON_MASTER,
        after_lt=last_lt,
        limit=100,
    )

    for tx in txs:
        if tx.direction != "in":
            continue
        if await _blob_exists(f"donations/processed/ton/{tx.hash}.json"):
            continue

        # 메모 파싱 — "JG-DON-{anon_short}" 형식만 허용
        memo = tx.comment or ""
        if not memo.startswith("JG-DON-"):
            await _queue_unmatched_donation(tx, chain="ton")
            continue
        anon_short = memo.replace("JG-DON-", "").strip()[:8]
        user_id = await _find_user_by_anon_short(anon_short)
        if not user_id:
            await _queue_unmatched_donation(tx, chain="ton")
            continue

        # 티어 매칭 + 포인트 지급
        await _process_donation(
            user_id=user_id, chain="ton", tx_hash=tx.hash,
            amount_usdt=tx.amount_usdt, from_address=tx.from_address,
        )

    if txs:
        await _save_last_processed_lt("ton_donation", txs[-1].lt)
```

### 4.2 TRC20 도네이션 검증

TRC20는 메모 미지원이므로 **사용자가 사전 등록한 출금 주소**로 매칭 (§7.5.1 of v2.0 spec과 동일).

### 4.3 티어 매칭 + 포인트 지급 (공통 로직)

```python
DONATION_TIERS = [
    # (USDT_min, USDT_max, multiplier, points)
    (1,    9,     3,  30),
    (10,   49,    3,  300),
    (50,   99,    4,  2_000),
    (100,  499,   7,  7_000),     # ⚡ 급격한 도약
    (500,  999,   10, 50_000),
    (1000, 2999,  12, 120_000),
    (3000, 4999,  14, 420_000),
    (5000, float("inf"), 15, 750_000),
]


def _resolve_donation_tier(amount_usdt: float) -> tuple[int, int, int]:
    """USDT 금액 → (멀티플라이어, 정확한 티어 USDT, 지급 P)"""
    for tier_min, tier_max, mult, points in DONATION_TIERS:
        if tier_min <= amount_usdt <= tier_max:
            # 정확한 티어 금액 미만 입금은 다음 낮은 티어로 fallback
            return mult, tier_min, points
    return 0, 0, 0


async def _process_donation(
    user_id: str, chain: str, tx_hash: str,
    amount_usdt: float, from_address: str,
) -> None:
    """도네이션 처리: 멱등성 보장 + 티어 매칭 + 포인트 지급"""
    # 멱등성 — tx_hash 중복 처리 방지
    blob_key = f"donations/processed/{chain}/{tx_hash}.json"
    if await _blob_exists(blob_key):
        return

    multiplier, tier_usdt, points = _resolve_donation_tier(amount_usdt)
    if points == 0:
        # 너무 작은 금액 (< $1)은 받지 않음 — 운영자 수동 처리 큐로
        await _queue_unmatched_donation(
            {"tx_hash": tx_hash, "user_id": user_id, "amount_usdt": amount_usdt},
            chain=chain, reason="below_minimum",
        )
        return

    donation = {
        "donation_id": str(uuid.uuid4()),
        "user_id": user_id,
        "chain": chain,
        "tx_hash": tx_hash,
        "from_address": from_address,
        "amount_usdt": amount_usdt,
        "tier_usdt": tier_usdt,
        "multiplier": multiplier,
        "points_granted": points,
        "received_at": datetime.utcnow().isoformat(),
        "status": "completed",
    }

    # 1. 도네이션 기록
    await _write_blob(f"donations/active/{user_id}/{donation['donation_id']}.json", donation)
    await _write_blob(blob_key, {"processed": True, "donation_id": donation["donation_id"]})

    # 2. 포인트 즉시 지급
    await _add_points(
        user_id, points,
        reason=f"donation_{chain}_{tier_usdt}",
        ref=donation["donation_id"],
    )

    # 3. 누적 도네이션 합계 업데이트 + 명예 등급 재계산
    profile = await _read_blob(f"users/{user_id[:2]}/{user_id}.json")
    profile["donation_total_usdt"] = profile.get("donation_total_usdt", 0) + amount_usdt
    profile["donation_count"] = profile.get("donation_count", 0) + 1
    profile["honor_label"] = _compute_honor_label(profile["donation_total_usdt"])
    await _write_blob(f"users/{user_id[:2]}/{user_id}.json", profile)

    # 4. Telegram 알림 (throttle 적용 — 도네이션은 무제한 발송 허용)
    await _send_telegram_message_throttled(
        user_id,
        f"💝 ${amount_usdt:.0f} USDT 후원 감사합니다.\n"
        f"+{points:,} P 즉시 지급되었습니다.\n"
        f"누적 후원: ${profile['donation_total_usdt']:.0f} · "
        f"명예 등급: {profile['honor_label']}",
        notification_key="donation_received",
        max_per_day=float("inf"),
    )

    # 5. 5만 P 이상 도네이션은 운영자에게도 알림 (감사 + 회계 추적)
    if points >= 50_000:
        await _notify_admin_large_donation(donation)


def _compute_honor_label(total_usdt: float) -> str:
    if total_usdt >= 5000: return "💎 Champion"
    if total_usdt >= 1000: return "🥇 Benefactor"
    if total_usdt >= 500:  return "🥈 Patron"
    if total_usdt >= 100:  return "💝 Supporter"
    return ""
```

### 4.4 입금 금액이 정확한 티어가 아닐 때

사용자가 $73 같은 어중간한 금액을 보내면:

| 입금 USDT | 적용 티어 | 지급 P | 사유 |
|---|---|---|---|
| $0.50 | — (미처리) | 0 | 최소 미달, 운영자 수동 처리 |
| $73 | $50 티어 | 2,000 P | 다음 낮은 티어로 fallback |
| $99 | $50 티어 | 2,000 P | $100 미달 |
| $100 | $100 티어 | 7,000 P | 정확 매칭 |
| $250 | $100 티어 | 7,000 P | $500 미달 → $100 티어 |
| $500 | $500 티어 | 50,000 P | 정확 매칭 |
| $7,500 | $5,000 티어 | 750,000 P | 최상위 티어 적용 |

> **주의**: 사용자가 $99를 보내면 $100 티어 보상 ($7,000 P) 대신 $50 티어 보상 ($2,000 P)을 받게 되어 **$5,000 P 손해**. 미니앱 UI에서 정확한 티어 금액을 권장 표시.

## 6. 어뷰징 방지

§11 of v2.0 spec의 일반 어뷰징 방어 위에 도네이션 특화 추가:

| 위협 | 대응 |
|---|---|
| **외부 지갑 도용으로 도네이션 후 환불 청구** | 도네이션은 환불 불가 약관 명시 + 입금 시 사용자가 미니앱 [동의] 체크 필수 |
| **자금 세탁 (큰 USDT를 도네이션 → 포인트 → 프리미엄 환매)** | Point는 비현금성, 외부 출금 불가 (§3.3 of v2.0 spec). 자금세탁 경로 차단 |
| **악의 사용자의 즉시 환불 요청 → 신고** | 도네이션 약관 + 한국 결제대행사 분쟁 정책 사전 검토 (§9 참조) |
| **메모 누락된 익명 입금** | `donations/unmatched/` 큐로 분리 → 운영자가 14일 이내 매칭 시도 → 매칭 실패 시 사용자 환불 옵션 (운영비 차감) |
| **테스트 트랜잭션으로 어뷰징 시도** | $1 미만 입금은 자동 처리하지 않음 + 동일 주소에서 24시간 내 5건 이상 도네이션 차단 |
| **단계적 큰 도네이션으로 KYC 우회** | $1,000+ 도네이션은 운영자 알림 + 7일 이내 KYC 절차 안내 (한국 자금세탁방지법 대응) |

## 7. 회계 처리

### 7.1 도네이션 매출 분류

| 항목 | 회계 분류 (한국 K-IFRS) | 세무 분류 |
|---|---|---|
| **도네이션 USDT 입금** | 매출 (서비스 후원 수익) | 부가가치세 면세? **세무 자문 필요** |
| **지급된 Point의 부채** | 충당금 (Point 소멸 시점까지) | — |
| **Point가 소멸 (1년 미사용)** | 부채 환입 → 추가 매출 | — |
| **Point가 프리미엄으로 전환** | 부채 차감 + 서비스 매출 인식 | — |

> 가상자산을 매출로 받는 회계 처리는 한국에서 아직 명확한 가이드가 없습니다. **MVP 출시 전 회계법인 1회 자문 필수** (예상 $500~1,500).

### 7.2 USDT → KRW 환전 시점

- 매월 말일 도네이션 누적분의 50%를 KRW 환전 (운영비 충당)
- 나머지 50%는 USDT로 보유 (시장 변동 헷지)
- 거래소: Upbit (Tether 직접 거래쌍 없으므로 BTC/USDT → KRW 우회) 또는 Bithumb

### 7.3 누적 도네이션 대시보드

## 9. 한국 규제 대응 (도네이션 특화)

§12 of v2.0 spec의 일반 규제 대응 위에 도네이션 특화:

| 규제 영역 | 적용 법령 | 대응 |
|---|---|---|
| **가상자산 사업자(VASP) 신고** | 특금법 7조 | 도네이션은 **환불 의무가 없으므로** 사용자 자산을 보관하지 않음 → VASP 미해당 가능. 단 변호사 자문 필수 |
| **유사 수신** | 자본시장법 + 유사수신규제법 | "이자 보장" 표현 금지. "후원에 대한 감사 포인트" 표현 사용 |
| **청약 철회권** | 전자상거래법 17조 | 디지털 콘텐츠 (Point)는 청약 철회 제외 가능 (시행령 21조). 단, 약관 명시 필수 |
| **부가가치세** | 부가세법 | 가상자산 매출의 부가세 처리는 명확하지 않음. 회계 자문 필수 |
| **외국환거래법** | 외환법 | TRC20 USDT는 미국 발행 → 외환 거래로 분류 가능. 한국은행 신고 필요 여부 검토 |
| **만 19세 미만 차단** | 게임산업진흥법 + 청소년보호법 | 미니앱 약관 동의 시 연령 자가 확인 (현행 텔레그램은 연령 검증 미지원) |
| **자금세탁방지** | 특금법 | $1,000+ 도네이션 시 KYC 절차 안내 + STR(의심거래) 보고 의무 검토 |

### 9.1 출시 전 변호사 자문 추가 질문 (도네이션 특화)

§12.1 of v2.0 spec의 5개 질문에 추가:

| # | 질문 |
|---|---|
| 6 | 환불 불가능한 USDT 도네이션이 가상자산 매매로 분류될 가능성? |
| 7 | 사용자가 사기 주장으로 환불 청구 시 운영자 책임 범위? |
